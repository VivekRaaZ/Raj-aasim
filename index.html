    Introduction

    JavaScript is a cross-platform, object-oriented scripting language. It is a small and lightweight language.

    JavaScript contains a standerd library of objects, such as Array, Date, and Math, and a core set of language elements such as operators, control structures, and statements.

    • Clint-side JavaScript extends the core language by supplying objects relevant to running JavaScript on a server.
    • Server-side JavaScript extends the core language by supplying objects relevant to running JavaScript on a server.

    JavaScript & Java

    JavaScript & Java are similar in some ways but fundamentallu different in some others.

    In contrast to Java's compile-time system of classes built by declarations, JavaScript Supports a runtime system based on a small number of data types represnting numeric, Boolean, and string values.

    Variables

    You use Variables as symbolic names for values in your appllication. The names of variables, called identifiers, conform to certain rules.

    A JavaScript identifier, must start with a letter, undercsore(_), or dollar sign($); subsequent characters can also be digits(0-9).

    You can ISO 8859-1 or Unicode letters such as a and u in identifiers.

    if...else statement
    Use the if statement to execute a statement if a logical condition is true. if(condition) { statement_1; }else { statement_2 } condition can be any expression that evaluates to true or false.

    You may also compoud the statements using else if to have multiple conditions tested in sequence, as follows:

    if(condition_1) { statement_1;} else if (condition_2) {statement_2} else if (condition_n) { statement_n }esle{ statement_last;} In the case of multiple condition only the first logical condition which evaluates to true will be executed. if ( condition ) { statement_1_runs_if_condition_is_true; statement_2_runs_if_condition_is_true; } else { statement_3_runs_if_condition_is_false; statement_4_runs_if_condition_is_false; } it is advisable to not use simple assignments in a conditionl expression, because the assignments can be condused with equality when glancing over the code. if((x=y)){/*statements here*/}
